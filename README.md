## 🚀 What's New in v0.15.4 (The "God Level" Update)

- **🔊 EAC3 / Dolby Audio Supported Out-of-the-Box** — No more "Audio format not supported" errors found in other major players. Enjoy high-quality cinematic sound flawlessly through nextlib's FFmpeg-based codec integration.
- **⚡ Context-Aware Playback Queue** — The player now intelligently syncs your playback queue based on where you started playing from (Folder, Playlist, Favorites, or All). Your queue always matches your context.
- **☑️ Bulk Selection (Audio & Video)** — Effortlessly manage your library by selecting and adding multiple files to Favorites or Playlists in a single tap. Multi-select for batch operations on both audio and video.
- **📂 Advanced Folder Navigation** — Re-engineered the music folder view. Clicking a folder now beautifully navigates *into* it for a cleaner, modern browsing experience instead of just expanding in place.

## 📡 Wi-Fi File Transfer

*Send and receive files between devices over Wi-Fi — no internet, no cables, no third-party apps.*

| Feature | Detail |
|:---|:---|
| 📡 **Local HTTP Server** | Start an HTTP server on your device to receive files from any browser |
| 📱 **QR Code Generation** | Auto-generate a QR code with the server URL and auth token |
| 📷 **QR Code Scanning** | CameraX + MLKit barcode scanning to quickly connect to another device |
| 📤 **Send Files** | Select videos or music, scan the receiver's QR code, and upload instantly |
| 🔑 **Auth Token Security** | Each session generates a unique auth token — no unauthorized uploads |
| 🛡️ **Path Traversal Protection** | File names are sanitized to prevent directory escape attacks |
| 🌐 **Cross-Platform** | Works with any device that has a browser — Android, iOS, PC, Mac, Linux |

### How It Works

```
┌──────────────────┐    QR Code / URL     ┌──────────────────┐
│   Device A       │ ──────────────────▶  │   Device B       │
│   (Sender)       │                      │   (Receiver)     │
│                  │    HTTP Upload        │                  │
│   Select Files   │ ──────────────────▶  │   HTTP Server    │
│   Scan QR        │    (Wi-Fi LAN)       │   :8080          │
│   Upload         │                      │   Save to /vault │
└──────────────────┘                      └──────────────────┘
```

## 🏗️ Architecture Deep Dive

<details>
<summary><b>🏗️ Click to expand — Architecture, Patterns & Data Flow</b></summary>

### Clean Architecture — Multi-Module

SHS Player follows **Clean Architecture** with strict layer separation across **11 Gradle modules**:

```
┌─────────────────────────────────────────────────────────────┐
│                      :app (Presentation)                     │
│    MainActivity · MusicScreen · MeScreen · TelegramScreen    │
│                  · PrivacyFolder · FileTransfer · QRScanner  │
├─────────────────────────────────────────────────────────────┤
│                   :feature (UI Layer)                        │
│   :feature:player     :feature:settings     :feature:videopicker│
│   PlayerActivity      9 Settings Screens     MediaPicker     │
│   PlayerViewModel     SettingsViewModels     MediaPickerVM   │
├─────────────────────────────────────────────────────────────┤
│                    :core (Domain Layer)                      │
│   :core:domain   :core:model   :core:common                  │
│   Use Cases       Data Models    Extensions/Utils            │
├─────────────────────────────────────────────────────────────┤
│                    :core (Data Layer)                         │
│   :core:data     :core:database    :core:datastore   :core:media│
│   Repositories   Room DB (v4)     DataStore Prefs   MediaStore│
├─────────────────────────────────────────────────────────────┤
│                    :core:ui (Design System)                   │
│   Theme · Components · Icons · Permissions · 40+ Translations│
└─────────────────────────────────────────────────────────────┘
```

### Key Architectural Patterns

| Pattern | Implementation |
|:---|:---|
| **MVVM** | ViewModels + StateFlow / collectAsStateWithLifecycle |
| **Hilt Dependency Injection** | @HiltAndroidApp, @AndroidEntryPoint, @Inject across all modules |
| **Repository Pattern** | Interface → Local/Fake implementations for testability |
| **Use Cases** | Domain layer with `GetSortedFoldersUseCase`, `GetSortedMediaUseCase`, etc. |
| **Kotlin Serialization** | For DataStore preferences and navigation route types |
| **Version Catalog** | Centralized dependency management via `gradle/libs.versions.toml` |
| **Room Schema Export** | 4 migration schemas stored for database evolution |
| **Global Crash Handler** | `GlobalExceptionHandler` → `CrashActivity` with logcat output |

### Data Flow

```
Preferences Flow:
DataStore → Serializer → DataSource → Repository → ViewModel → UI

Media Flow:
MediaStore ContentProvider → MediaService → MediaSynchronizer → Room DB → Repository → UseCase → ViewModel → UI

Player Flow:
Intent → PlayerActivity → MediaController (Media3 Session) → PlayerService → ExoPlayer + NextLib
```

### Database Schema (Room — 4 Migrations)

| Entity | Purpose |
|:---|:---|
| `MediumEntity` | Video/audio file metadata |
| `MediumStateEntity` | Per-video playback state (position, audio track, subtitle track, speed) |
| `DirectoryEntity` | Folder information for library browsing |
| `FavoriteEntity` | Favorite media items |
| `BookmarkEntity` | Per-video timestamp bookmarks |
| `VideoStreamInfoEntity` | Video codec, resolution, bitrate, frame rate |
| `AudioStreamInfoEntity` | Audio codec, channels, sample rate, language |
| `SubtitleStreamInfoEntity` | Subtitle codec, language, embedded/external |
| Relations | `DirectoryWithMedia`, `MediumWithInfo` |
